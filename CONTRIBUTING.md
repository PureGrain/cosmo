# Contributing to Cosmo

Thanks for your interest in contributing! Here's how to get started.

## Getting Started

1. Fork the repo and clone locally
2. Follow the [Quick Start](README.md#quick-start) to get a dev environment running
3. Create a feature branch: `git checkout -b my-feature`
4. Make your changes
5. Test locally (frontend + API)
6. Commit with a clear message
7. Push and open a PR

## Guidelines

- **Keep it simple.** Cosmo's strength is zero frameworks and minimal dependencies. Don't add frameworks or build tools.
- **No new dependencies** without discussion. Open an issue first if you think a dependency is needed.
- **Test your changes.** Run the API and frontend locally, verify chat works, check both light and dark mode.
- **One PR per feature/fix.** Keep PRs focused and reviewable.

## What We're Looking For

- Bug fixes
- Documentation improvements
- Accessibility improvements
- New prompt templates or example personas
- Performance optimizations
- Security improvements

## Code Style

- Vanilla JS (ES5-compatible in frontend for broad support)
- No TypeScript, no JSX, no build step
- 2-space indentation
- Single quotes in JS
- Semicolons required

## Reporting Issues

- Use GitHub Issues
- Include: what you expected, what happened, browser/OS, and steps to reproduce
- For security vulnerabilities, email the maintainer directly (don't open a public issue)
