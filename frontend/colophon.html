Architecture

This site uses a clean split architecture: a static frontend served from a CDN, backed by a lightweight API server that handles the chatbot, contact form, and analytics.

The Chatbot

The AI chatbot on this site isn't a generic widget. It's a custom-built conversational agent powered by Claude (Anthropic's AI). It has access to a detailed knowledge base covering career history, projects, skills, and services.

  • Streaming responses: Uses Server-Sent Events (SSE) for real-time, token-by-token output
  • Context-aware: The system prompt includes the full resume, project details, and conversation history
  • Guardrails: Deflection rules prevent the bot from discussing topics outside the portfolio scope
  • Markdown rendering: Responses are rendered with proper formatting, code blocks, and links
  • Configurable personality: Bot name, tone, and suggestions are all defined in the config file

Frontend

The entire frontend is static HTML, CSS, and vanilla JavaScript. No React, no build step, no framework. Every page is config-driven from a single JavaScript file.

  • Config-driven: One config file controls all content, branding, and navigation
  • Dark/light theme: System-aware theme toggle with smooth transitions and persistent preference
  • Responsive: Mobile-first layout that works on all devices
  • Performance: No bundler, no virtual DOM, no hydration. Just fast, cacheable static files
  • Accessibility: Semantic HTML, keyboard navigation, and proper color contrast

API Server

A minimal Node.js + Express server handles three responsibilities: the chatbot, the contact form, and visitor analytics. Designed for single-instance deployment with low resource usage.

  • Chat endpoint: Proxies requests to the Claude API with system prompt injection and conversation management
  • Contact endpoint: Validates input, checks for spam patterns, and forwards messages via email
  • Rate limiting: Per-IP limits on chat and contact endpoints to prevent abuse
  • CORS: Whitelist-based origin control so only the portfolio domain can call the API
  • Stateless: No database required. Conversations live in memory for the session duration only

CI/CD Pipeline

Deployment is automated through a standard continuous delivery pipeline. Push to main, and the site updates.

  • Frontend: Deployed to a CDN (e.g., Cloudflare Pages, Netlify, or Vercel) on every push to the main branch
  • API: Deployed to a server or container host with automatic restart on failure
  • Environment: Secrets managed via environment variables, never committed to the repo
  • Preview: Branch deployments create preview URLs for testing before merge

Visitor Analytics

Lightweight, privacy-respecting analytics are collected server-side. No third-party tracking scripts, no cookies, no fingerprinting.

  • Page views: Tracked server-side via a simple beacon endpoint
  • Chat analytics: Conversation count, message count, and popular questions (no content stored)
  • No PII: IP addresses are hashed, not stored. No cookies are set for tracking
  • Dashboard: Analytics data is available through the API for the site owner

Security Stack

Multiple layers of protection keep the site and its visitors safe.

  • Bot protection: Cloudflare Turnstile (optional) on contact forms to block spam without CAPTCHAs
  • Honeypot detection: Hidden form fields catch automated bots that fill every input
  • Rate limiting: Per-IP request limits on all API endpoints
  • Input sanitization: All user input is validated and sanitized before processing
  • HTTPS everywhere: TLS enforced on both frontend and API
  • Content Security Policy: Strict CSP headers limit what scripts and resources can load
